Comms design system — v1 editorial (paper + ink + ember)

Single color preview for the v1 design system. Canonical tokens live in tokens/design-tokens.json and tokens/tokens.css. All colors are HSL triplets consumed as hsl(var(--token)).

Runtime semantic mapping

  • Primary / ink: --primary hsl(222 30% 14%) — deep dark blue for primary actions, links, default interactive.
  • Ember: --ember hsl(14 80% 48%) — warm orange-red for priority cues, "what changed," urgency emphasis.
  • Signal positive: --signal-positive hsl(152 45% 32%) — resolved / stable / low-drama state.
  • Signal warning: --signal-warning hsl(36 85% 42%) — time-bound escalation / pay-attention-soon.
  • Destructive: --destructive hsl(0 70% 42%) — highest urgency, irreversible actions.
  • Background / paper: --background hsl(38 30% 96%) — warm off-white page surface.
  • Foreground / ink: --foreground hsl(222 25% 12%) — default text.

Why these decisions

  • Paper + ink metaphor: warm background and dark ink foreground evoke editorial newsroom legibility under monitoring load.
  • Ember as single emphasis accent: a single warm orange-red for "what changed" prevents attention fragmentation from multiple competing accent hues.
  • Primary (ink) vs ember: deep ink handles actions/navigation; ember handles urgency/priority — never mixed to avoid misreading an action as an alert.
  • Signal positive is intentionally calm: green stays muted to avoid visual parity with ember; success should feel resolved, not excited.
  • Signal warning and destructive separation: amber vs red — clearly distinct at small badge/rail sizes.

Note: keep icon + label with status colors; do not rely on color alone.
