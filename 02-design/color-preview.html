Comms design system — final color decisions

This is the single color preview file for the v0 design system. Canonical tokens live in tokens/design-tokens.json.

Final semantic mapping

  • Accent: #4F6887 (primary actions, links, new-since-last-check cues).
  • Warning: #D15400 (time-bound escalation / pay-attention-soon).
  • Critical: #922B2E (highest urgency, immediate response path).
  • Success: #68874F (resolved / stable / low-drama state).
  • Highlight: #CEBA71 (non-urgent notice; use as fill with dark text).
  • Text primary: #454545.
  • Secondary purple: removed from the UI token schema.

Why these decisions

  • Accent + warning separation: blue and orange are clearly distinct for scanning.
  • Critical red differs from warning orange: avoids merge at small badge/rail sizes.
  • Highlight is not warning: yellow is a “look here” cue, not escalation by itself.
  • Single non-status hue strategy: removing purple keeps the system quieter and easier to parse under monitoring load.
  • Text pairing: white on accent (~5.73:1), accent on white (~5.73:1), primary text on white (~9.59:1).

Note: success green #68874F is intentionally calm. Keep icon + label with status colors; do not rely on color alone.
